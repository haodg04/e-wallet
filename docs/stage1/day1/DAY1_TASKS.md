# Day 1 (May 20) - Tasks & Deliverables

## 🎯 Focus: Yêu cầu & MVP Chốt

### Morning (9:00 - 12:00)

#### Task 1.1: Review yêu cầu nghiệp vụ từ stakeholders
- **Owner**: PM
- **Duration**: 2h
- **Steps**:
  1. Họp với team để xác nhận scope (1h)
  2. Làm rõ target users, pain points, unique selling points (30 phút)
  3. Ghi lại constraints: bảo mật, throughput, compliance (30 phút)
- **Deliverable**: REQUIREMENTS.md hoàn tất
- **Acceptance**: Tất cả stakeholder requirements được documented

#### Task 1.2: Chốt MVP - Features bắt buộc vs. Future
- **Owner**: PM + Tech Lead
- **Duration**: 1h
- **Steps**:
  1. Review danh sách đầy đủ từ Stakeholder
  2. Lọc theo "must-have", "nice-to-have", "future"
  3. Viết acceptance criteria cho mỗi feature MVP
- **Deliverable**: MVP.md hoàn tất
- **Acceptance**: 10 core features locked, no ambiguity

### Afternoon (13:00 - 17:00)

#### Task 1.3: Security & Performance requirements
- **Owner**: Tech Lead + Security
- **Duration**: 2.5h
- **Steps**:
  1. Định nghĩa SLA: API latency, uptime, throughput (1h)
  2. Danh sách bảo mật: encryption, auth, rate limit (1h)
  3. Ghi vào REQUIREMENTS.md phần "Non-functional" (30 phút)
- **Deliverable**: REQUIREMENTS.md NFR section updated
- **Acceptance**: All SLA & security requirements written

#### Task 1.4: Tạo ban sơ Gantt chart (5 ngày + 2 Sprint sau)
- **Owner**: PM
- **Duration**: 1.5h
- **Steps**:
  1. Ghi vào file PHASE1_TIMELINE.md
  2. Phân bổ task theo ngày hôm nay
  3. Identify blockers & dependencies
- **Deliverable**: PHASE1_TIMELINE.md tạo
- **Acceptance**: Clear timeline for next 4 days
